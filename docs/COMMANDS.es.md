# Comandos

Referencia rápida de comandos para desarrollo y operación.

## Uso recomendado
- Instalar dependencias del proyecto.
- Ejecutar backend y frontend en desarrollo.
- Ejecutar lint, typecheck y tests antes de validar cambios.

## Flujo diario sugerido
- Arrancar stack de desarrollo (backend + frontend).
- Ejecutar verificación local antes de push.
- Validar capturas/documentación cuando hay cambios UX.

## Flujo de calidad
- Lint + typecheck.
- Tests unitarios y e2e según impacto.
- Revisión manual de vistas críticas (admin/jugador/anónimo).

## Notas
- Usa el archivo canónico para comandos exactos y flags.
