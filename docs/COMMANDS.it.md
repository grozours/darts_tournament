# Comandi

Riferimento rapido dei comandi principali per sviluppo e operazioni.

## Uso consigliato
- Installare le dipendenze del progetto.
- Avviare backend e frontend in sviluppo.
- Eseguire lint, typecheck e test prima della validazione.

## Flusso quotidiano consigliato
- Avviare stack sviluppo (backend + frontend).
- Eseguire controlli locali prima del push.
- Aggiornare screenshot/documentazione quando cambia UX.

## Flusso qualità
- Lint + typecheck.
- Test unitari ed e2e in base all’impatto.
- Verifica manuale delle viste critiche (admin/giocatore/anonimo).

## Nota
- Per comandi e opzioni esatti, usare la documentazione canonica EN/FR.
