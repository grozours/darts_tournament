# Comandos

Referência rápida dos comandos principais para desenvolvimento e operação.

## Uso recomendado
- Instalar dependências do projeto.
- Iniciar backend e frontend em modo de desenvolvimento.
- Executar lint, typecheck e testes antes da validação.

## Fluxo diário recomendado
- Iniciar stack de desenvolvimento (backend + frontend).
- Executar verificação local antes de push.
- Atualizar screenshots/documentação quando houver mudança de UX.

## Fluxo de qualidade
- Lint + typecheck.
- Testes unitários e e2e conforme impacto.
- Revisão manual de vistas críticas (admin/jogador/anónimo).

## Nota
- Para comandos e flags exatos, usar a documentação canónica EN/FR.
